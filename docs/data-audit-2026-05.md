## 一、核心结论（先看这段）

当前官网展示的核心数字与负责人底表存在**两个层面的偏差**：第一，**生态卡 SciBase 的"25M+ OA 文献"和数据能力面板的"25M+ OA 文献"严重低估**，真实底表 OA 量为 1.25 亿；学术文献总量更是高达 3.41 亿，与当前 25M+ 的口径至少差一个数量级。第二，**数据能力面板的"570K+ 蛋白注释 / 10M+ 化学反应 / 50K+ 教材"与 Sciverse 平台主定位（"AI-Ready 学术知识底座"）**不在同一抽象层：化学反应、蛋白注释属于 DianShi/SeqStudio 子产品的纵深能力，与"全平台知识规模"混排会让访客误以为 Sciverse 总量只有千万级。

简言之：**数字不是太多，而是用错了位置**。建议将"全平台规模数字"上提到 Sciverse 主面板，子产品纵深数字下沉到对应卡片，二者不再混用。

## 三、可信度与定位匹配度判断

### 1. 哪些"值得放"且"匹配 AI-Ready 知识底座"定位
Sciverse 的核心定位是"面向 Agent 的科学知识底座"，最能直接传递这层定位的数字是**总量、AI-Ready 占比、覆盖广度**三类。底表中 5.16 亿条知识记录、1.02 亿篇 AI-Ready 全文、1.25 亿篇 OA、3.41 亿篇文献、1.05 亿册图书、约 7000 万专利、814 种语言、Nature / Science / Cell 三大出版商 159 种顶刊 98% 覆盖率，全部是直接证据，应优先放在最显眼的"数据能力 全景"区。

### 2. 哪些"值得放"但要降级到子产品卡
化学反应 10M+、化学物质 6M+、蛋白注释 570K+、SA-RxnDiagram 15k、BLAST/Foldseek 一体化等都是**纵深能力**，应当且只应当出现在 DianShi / SeqStudio / Sci-Base 三张卡里，不应当与 Sciverse 全平台数字并排展示。当前数据能力面板把"化学反应 10M+"和"全平台 OA 25M+"放在同一行，不仅造成数量级错乱，更让访客误以为这就是 Sciverse 的天花板。

### 3. 哪些"不建议放"
当前数据能力面板的"23 维 functional axes"过于专业且只属于 SeqStudio，对官网首屏访客无意义，且占用了一个本可以用于"语言覆盖 / 顶刊覆盖 / AI-Ready 占比"的高价值位置。三宫格里的"2,000 万+ 元数据"与底表口径不符（底表 5.16 亿），明显低估，应直接替换为更具说服力的数字。

## 五、可放进未来"数据深度"二级页的高价值数字

负责人底表里还有一批数字暂时不适合放在首屏，但极适合做成一个"数据深度"或"覆盖范围"的二级 landing 页（与"接入指南"平级），未来可考虑：

- Nature 91 + Science 22 + Cell 46 = 159 顶刊 / 145.6 万篇 / 98.03% 覆盖率
- AI 子领域专题（人工智能 591 万 / CV 159 万 / LLM 10.3 万 / Transformer 16.5 万）
- AI 顶会顶刊覆盖（CVPR 6,211 / ACL 8,863 / ICML 2,133 / EMNLP 872 …）

这些数字共同构成 Sciverse "广度可信"和"深度可信"的双层证据，建议作为下一阶段官网内容路线图的核心素材。
